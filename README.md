# Say It Human

让文字像人一样说话。

## 简介

Say It Human 致力于消除 AI 味和套话，让内容更自然、更真实。

本项目包含两个 Claude Code Skills：

- **去AI味**：AI写的，改成人说的
- **改稿手**：人写的，找到他真正想说的

## 使用指南

| 你的内容 | 用哪个 | 核心动作 |
|---------|--------|---------|
| AI生成的，机械、空洞 | 去AI味 | 概念→场景，趋势→时间线 |
| 人写的，但全是套话 | 改稿手 | 找到真话，去除废话 |

## 去AI味

**唯一边界**：只用原文素材，不编人物、不造故事、不加细节。

**5类改写示例**：

**观点类** —— 删掉"至关重要""值得注意的是"，压成有立场的判断
> ❌ "提示词工程已经成为一项至关重要的技能。它不仅仅是简单地向AI提问，而是一门关于如何与AI进行高效沟通的艺术。"
> ✅ "提示词不是技能，是习惯。区别不在会不会用AI，在会不会想清楚自己要什么。"

**方法论类** —— 打破"首先…其次…此外…最终"，只打一个点
> ❌ "要提高团队的工作效率，首先需要建立清晰的目标管理体系。其次，要注重团队成员之间的沟通与协作。此外，定期的复盘和总结也是不可或缺的环节。"
> ✅ "团队效率低，十次有八次不是能力问题，是不知道在干嘛。目标说不清，干活全靠猜。"

**趋势分析类** —— 删除"随着…发展""值得思考"，用类比替代论证
> ❌ "随着人工智能技术的不断发展和普及，内容创作领域正在经历深刻的变革。传统的内容创作模式面临着前所未有的挑战。"
> ✅ "AI时代创作者的核心竞争力不是写作能力，是肚子里有没有货。AI是放大器，不是发电机。"

**产品/功能介绍类** —— 砍掉宣传词汇，换成说话语气
> ❌ "我们全新推出的智能助手具备多项强大功能。它不仅能够高效地处理日常任务，还能基于用户的使用习惯提供个性化的建议和推荐。"
> ✅ "这个助手能干嘛？你常做的事它帮你做，你没想到的事它提醒你。说人话就能用，不用学。"

**总结复盘类** —— 删除套话，数据保留，口号变行动
> ❌ "回顾过去一年的工作，我们取得了显著的成绩。在团队的共同努力下，项目按时交付率提升了30%。展望未来，我们将继续秉承创新理念，携手共进，再创佳绩！"
> ✅ "去年干得不错：交付准时率涨了30%。但人走了不少，市场也更卷了。今年就两件事：留住人，继续交付。"

## 改稿手

**核心目标**：改完之后，这篇文章里要有一个真实的人在说一件真实的事。

**动笔前判断三件事**：
1. 这个人是谁？（管理者、技术人员、普通职场人？）
2. 他在跟谁说话？（内部汇报、对外发布、说服某个人？）
3. 剥掉所有修饰，他真正要说的是哪几句？

**改稿决策表**：

| 情况 | 处理方式 |
|------|----------|
| 意思在，但被套话埋住了 | **大改** —— 挖出来 |
| 意思在，但表达方式不像这个人 | **轻改** —— 调整表达 |
| 找不到真实的人在说一件真实的事 | **不改** —— 反问并给方向 |
| 整篇就是仪式性文件 | **不挖真话** —— 只做清理 |
| 本来就是他会说的话 | **不动** |

**改稿纪律**：
- 不加事实，不编细节
- 不改立场，不改程度（"有一定难度"不能变成"根本不行"）
- 保留专业术语、行话、口头禅
- 砍的是套话，不是专业性

**什么该砍**：
- 套话开头："在当今…""随着…的发展"
- 程度虚词："至关重要""不可或缺""显著提升"
- 虚假对仗："不是A而是B"——凑出来衬托的砍
- 口号收尾："让我们共同…""未来可期"
- 强行清单："首先…其次…此外…最终"

**节奏原则**：真人写东西不是每句都在发力。有时说完再补一句，有时自我修正。允许有松的地方，不要把每句都改成金句。

## 示例对比

### 去AI味示例

**输入**：
```
坐落于繁华都市的一隅，这家独立书店以其独特的选书品味和温馨的阅读氛围，成为城市文化爱好者的精神栖息地。我们致力于为每一位读者提供沉浸式的阅读体验，让知识与生活完美交融。
```

**输出**：
```
这家店藏在巷子深处，门面不大，但书选得很有意思。老板是个退休的中学语文老师，每周三下午会在店里给人讲古诗。来的人不多，但都是常客。
```

### 改稿手示例

**输入**：
```
本周在各位同事的通力协作下，项目取得了阶段性进展。我们克服了诸多困难，按计划完成了核心功能的开发工作。同时，我们也清醒地认识到存在的不足，将在后续工作中持续改进。
```

**输出**：
```
这周干了三件事：
- 用户登录模块做完，测了没大问题
- 支付接口对接遇到坑，对方的文档写得不清楚，耽误了一天
- 周五开了评审会，产品经理又改了需求

下周重点：把支付流程跑通，争取提测。
```

## 安装方法

### 手动安装

1. 下载或克隆本仓库
2. 将 `skills/` 文件夹复制到 Claude Code 的 skills 目录：
   - macOS/Linux: `~/.claude/skills/`
   - Windows: `%USERPROFILE%\.claude\skills\`

3. 目录结构：
```
~/.claude/skills/
├── humanize-ai/
│   └── SKILL.md
└── editor-revisor/
    └── SKILL.md
```

### Git 克隆

```bash
git clone https://github.com/taxue2025/say-it-human.git
cp -r say-it-human/skills/* ~/.claude/skills/
```
